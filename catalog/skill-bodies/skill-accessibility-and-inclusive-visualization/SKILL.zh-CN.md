# 无障碍与包容性可视化

## 概述

当可视化必须让更多人、在更多情境下、面对更多辅助需求时都能理解，请使用此技能。无障碍不是后处理步骤；它会影响图表选择、颜色、标注、交互、后备文本和导出策略。

默认假设：每个重要可视化都应提供一条非视觉路径来传达关键洞察，无论是通过周围的文字、直接标签、数据表，还是正式的文本替代方案。

## 工作模式

1. 确定图表是探索型、解释型、交互型还是导出型。
2. 决定哪些信息在没有悬停、颜色辨别、精确指针操作、展开面板、私有持久化状态、受权限限制的能力或稳定网络连接时仍必须可用。
3. 如果叙事使用生成式图像、插图、WebGL、粒子、3D、地图、滚动叙事、视差或动画，应将素材所展示的内容与数据所证明的内容区分开来。
4. 如果曾使用 Codex 图像生成功能构思布局、图形、页面集成、素材或关键帧概念，请验证：已向用户展示大屏和移动端概念图，并附有简洁的规划与交互要点；在项目变更或实现代码开始之前，用户已批准所生成的设计集；且 `../../references/foundations/meaning-preserving-visual-design-workflow.md` 中的语义设计契约已经存在：无障碍路径必须保留与视觉设计相同的论点、限定条件、来源上下文、证据层级、锁定布局元素、移动端延续方式和交互含义。
5. 如果视图是类 UML、ERD、状态机、工作流、依赖关系或架构图，请保留节点、分组、关系和所选路径的文本大纲；使用 `../uml-and-software-architecture-visualization/SKILL.md` 获取图示专用语义。
6. 使用 `../../references/foundations/mobile-first-responsive-visualization.md` 验证触控目标、拖拽替代方案、键盘打开时的可视视口行为、主要可视化的可见性、网络不稳定状态，以及 AR、相机、运动、振动、通知和地理位置功能的后备方案。
7. 提供直接标签、强对比度、冗余编码、键盘操作路径、减少动态替代方案、无障碍披露控件和文本替代方案。
8. 对于交互式可视化，检查共享 URL、保存的视图、刷新以及前进/后退导航是否会保留与视觉界面相同的无障碍状态摘要。
9. 同时测试图表或图示及其周围的叙述内容。

## 输出要求

- 指明该图表特有的无障碍风险，而不只是列举通用 WCAG 条目。
- 为屏幕阅读器、PDF 和静态导出提供后备策略。
- 说明在不依赖悬停或仅凭颜色的情况下，哪些内容必须保持可见。
- 说明当配置或下钻面板折叠时，哪些活动筛选器、选择项、限定条件和汇总值仍保持可见。
- 对于颜色指导，请说明语义颜色角色、对比度风险、冗余编码，以及灰度或色觉缺陷审查路径。
- 对于图像辅助的视觉叙事，提供替代文本或长描述，涵盖场景、数据层、主要结论和限定条件，且不夸大生成式图像所表达的内容。
- 对于经过概念设计的可视化，请确保存在用户批准记录，并确保长描述、减少动态路径、静态导出及来源/限定条件文本都保留已批准的语义设计契约和锁定的概念元素，而不只是最终渲染的像素。
- 对于移动端可视化，请确保主要证据不会被设置内容挤到下方；触控目标和命中区域足够大；拖拽操作有替代方式；悬停操作有轻触/焦点等效方式；键盘打开状态仍可操作；陈旧/离线状态有明确说明；传感器/相机/通知功能有无需权限的后备方案。
- 对于动画，请指定减少动态行为及关键帧或最终状态后备方案。
- 对于滚动叙事或视差效果，请保留原生滚动和键盘行为，指定静态关键帧或堆叠式后备方案，并确保禁用动态效果时不会丢失证据。
- 对于 WebGL 或粒子效果，请用文本描述数据含义，为重要标记提供可通过键盘访问的焦点或选择路径，并提供保留流动、方向、焦点和不确定性的非动画后备方案。
- 对于交互式图示，请确保搜索、选择、详情面板、重置、导出、展开/折叠和下钻均可在不依赖仅指针交互的情况下访问。
- 对于可共享的可视化，请确保复制链接、保存视图、重置、展开/折叠面板以及恢复的 URL 状态对键盘和屏幕阅读器用户而言都可访问且可理解。
