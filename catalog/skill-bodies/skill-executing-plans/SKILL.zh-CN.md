# 执行计划

## 概述

加载计划，严格审查，执行所有任务，完成后汇报。

**开始时宣布：**“我正在使用 executing-plans 技能实施此计划。”

**注意：** 告知人类协作者，Superpowers 在能够使用 subagents 时效果要好得多。如果在支持 subagent 的平台（例如 Claude Code 或 Codex）上运行，工作质量会显著提升。如果 subagents 可用，请使用 superpowers:subagent-driven-development，而不是此技能。

## 流程

### 第 1 步：加载并审查计划
1. 阅读计划文件
2. 严格审查——找出对计划的任何问题或疑虑
3. 如果存在疑虑：开始前向人类协作者提出
4. 如果不存在疑虑：创建 TodoWrite 并继续

### 第 2 步：执行任务

对于每项任务：
1. 标记为 in_progress
2. 严格遵循每个步骤（计划中的步骤均为小粒度步骤）
3. 按要求运行验证
4. 标记为 completed

### 第 3 步：完成开发

所有任务均已完成并通过验证后：
- 宣布：“我正在使用 finishing-a-development-branch 技能完成这项工作。”
- **必需的子技能：** 使用 superpowers:finishing-a-development-branch
- 按照该技能验证测试、呈现选项并执行选择

## 何时停止并寻求帮助

**遇到以下情况时立即停止执行：**
- 遇到阻碍（缺少依赖项、测试失败、说明不清楚）
- 计划存在妨碍启动的关键缺口
- 你不理解某条说明
- 验证反复失败

**请求澄清，不要猜测。**

## 何时回到较早步骤

**在以下情况下返回审查（第 1 步）：**
- 协作者根据你的反馈更新了计划
- 基本方法需要重新思考

**不要强行越过阻碍**——停下来询问。

## 请记住
- 首先严格审查计划
- 严格遵循计划步骤
- 不要跳过验证
- 计划要求引用技能时，务必引用
- 受阻时停止，不要猜测
- 未经用户明确同意，绝不在 main/master 分支上开始实施

## 集成

**必需的工作流技能：**
- **superpowers:using-git-worktrees** - 确保使用隔离的工作区（创建一个工作区或验证现有工作区）
- **superpowers:writing-plans** - 创建此技能所执行的计划
- **superpowers:finishing-a-development-branch** - 在所有任务完成后完成开发工作
