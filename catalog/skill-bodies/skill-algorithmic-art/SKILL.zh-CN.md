算法哲学是通过代码表达的计算美学运动。输出 .md 文件（哲学）、.html 文件（交互式查看器）和 .js 文件（生成算法）。

此过程分为两步：
1. 创建算法哲学（.md 文件）
2. 通过创建 p5.js 生成艺术来表达（.html + .js 文件）

首先执行以下任务：

## 创建算法哲学

首先创建一种算法哲学（而不是静态图像或模板），它将通过以下方式得到诠释：
- 计算过程、涌现行为、数学之美
- 带种子的随机性、噪声场、有机系统
- 粒子、流动、场、力
- 参数化变化与受控混沌

### 必须理解的关键点
- 收到什么：用户给出一些需要考虑的细微输入或指令，但只把它们当作基础；它们不应限制创作自由。
- 创造什么：一种算法哲学/生成美学运动。
- 接下来发生什么：同一版本接收该哲学并通过代码表达它——创建由 90% 算法生成和 10% 必要参数构成的 p5.js sketch。

可以考虑以下方法：
- 为一种生成艺术运动撰写宣言
- 下一阶段编写让它鲜活起来的算法

该哲学必须强调：算法表达、涌现行为、计算之美、带种子的变化。

### 如何生成算法哲学

**为该运动命名**（1–2 个词）：“有机湍流”/“量子谐波”/“涌现静谧”

**阐明哲学**（4–6 段——简洁但完整）：

为捕捉算法本质，请表达该哲学如何通过以下方面显现：
- 计算过程与数学关系？
- 噪声函数与随机模式？
- 粒子行为与场动力学？
- 时间演化与系统状态？
- 参数化变化与涌现复杂性？

**关键指南：**
- **避免重复**：每个算法方面只应提及一次。除非增加新的深度，否则不要重复噪声理论、粒子动力学或数学原理等概念。
- **反复强调工艺水准**：该哲学必须多次强调，最终算法应给人一种历经无数小时开发、经过精心打磨、出自领域顶尖专家之手的感觉。这种表述至关重要——反复使用“精心打造的算法”“深厚计算专业能力的结晶”“不遗余力的优化”“大师级实现”等措辞。
- **留出创作空间**：明确算法方向，但要足够简洁，让下一个 Claude 有空间以极高的工艺水准作出诠释性的实现选择。

该哲学必须引导下一个版本以算法方式表达想法，而不是通过静态图像。美存在于过程中，而不是最终画面中。

### 哲学示例

**“有机湍流”**
哲学：受自然法则约束的混沌，从无序中涌现秩序。
算法表达：由分层 Perlin 噪声驱动的流场。数千个粒子跟随向量力，其轨迹累积成有机密度图。多个噪声八度形成湍流区域和平静地带。颜色从速度和密度中涌现——快速粒子明亮燃烧，慢速粒子隐入阴影。算法持续运行直至达到平衡——这是一种精心调校的均衡，每个参数都由计算美学大师经过无数次迭代打磨。

**“量子谐波”**
哲学：离散实体呈现波状干涉模式。
算法表达：粒子初始化于网格上，每个粒子携带一个通过正弦波演化的相位值。当粒子彼此靠近时，相位发生干涉——相长干涉形成明亮节点，相消干涉形成空洞。简单谐振运动生成复杂的涌现曼荼罗。结果来自不遗余力的频率校准，每个比例都经过精心选择，以创造共振之美。

**“递归低语”**
哲学：跨尺度的自相似，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支都有轻微随机性，但受黄金比例约束。L-system 或递归细分生成兼具数学感和有机感的树状形态。细微噪声扰动打破完美对称。线宽随每层递归逐渐减小。每一个分支角度都是深入数学探索的成果。

**“场动力学”**
哲学：通过不可见力对物质产生的作用，使其变得可见。
算法表达：由数学函数或噪声构建向量场。粒子诞生于边缘，沿场线流动，在达到平衡或边界时消亡。多个场可以吸引、排斥或旋转粒子。可视化只展示轨迹——如幽灵般证明不可见力的存在。这是一支通过力的平衡而精心编排的计算舞蹈。

**“随机结晶”**
哲学：随机过程结晶为有序结构。
算法表达：随机圆形堆积或 Voronoi 镶嵌。从随机点开始，让它们通过松弛算法演化。单元彼此推开，直到达到平衡。颜色取决于单元大小、邻居数量或与中心的距离。涌现的有机铺砌既随机又必然。每个种子都会产生独一无二的晶体之美——这是大师级生成算法的印记。

*这些是压缩后的示例。实际算法哲学应包含 4–6 个内容充实的段落。*

### 基本原则
- **算法哲学**：创建一种通过代码表达的计算世界观
- **过程优先于结果**：始终强调美从算法执行中涌现——每次运行都独一无二
- **参数化表达**：思想通过数学关系、力和行为来传达——而不是静态构图
- **艺术自由**：下一个 Claude 以算法方式诠释该哲学——为创造性实现留出空间
- **纯粹生成艺术**：目标是创造鲜活的算法，而不是带有随机性的静态图像
- **专家级工艺**：反复强调最终算法必须给人精心制作、历经无数次迭代打磨的感觉，是计算美学领域顶尖专家深厚能力的结晶

**算法哲学应为 4–6 段。** 用富有诗意的计算哲学充实它，将预期愿景融为一体。避免重复相同观点。将此算法哲学输出为 .md 文件。


**要设计有效参数，应关注系统需要可调的属性，而不是从“模式类型”的角度思考。**

**核心算法——表达哲学**：

**关键**：算法哲学应决定要构建什么。

要通过代码表达哲学，不要想“我应该使用哪种模式？”，而要想“如何通过代码表达这种哲学？”

如果哲学关乎**有机涌现**，可以考虑使用：
- 随时间累积或生长的元素
- 受自然规则约束的随机过程

如果哲学关乎**数学之美**，可以考虑使用：
- 几何关系与比例
- 三角函数与谐波
- 创造意外模式的精确计算

如果哲学关乎**受控混沌**，可以考虑使用：
- 严格边界内的随机变化
- 分岔与相变
- 从无序中涌现秩序

**算法源自哲学，而不是来自选项菜单。**

### 工艺水准要求

**关键**：为了达到大师水准，应创建给人以生成艺术大师历经无数次迭代打磨之感的算法。仔细调校每个参数。确保每种模式都带着明确目的涌现。这不是随机噪声——而是经深厚专业能力锤炼的受控混沌。

## 创建交互式制品

**提醒：此时应已阅读 `templates/viewer.html`（参见步骤 0）。以该文件作为起点。**

为了让用户探索生成艺术，请创建一个单一、自包含的 HTML 制品。确保该制品可立即在 claude.ai 或任何浏览器中运行——无需设置。将所有内容以内联方式嵌入。

### 关键：哪些固定，哪些可变

`templates/viewer.html` 文件是基础。它包含所需的确切结构和样式。

**固定（始终完全按原样包含）：**
- 布局结构（页眉、侧边栏、主 Canvas 区域）
- Anthropic 品牌设计（UI 色彩、字体、渐变）
- 侧边栏中的 Seed 部分：
  - 种子显示
  - Previous/Next 按钮
  - Random 按钮
  - 跳转到种子的输入框 + Go 按钮
- 侧边栏中的 Actions 部分：
  - Regenerate 按钮
  - Reset 按钮

**可变（为每件艺术作品定制）：**
- 整个 p5.js 算法（setup/draw/class）
- 参数对象（定义艺术作品需要什么）
- 侧边栏中的 Parameters 部分：
  - 参数控件数量
  - 参数名称
  - 滑块的 min/max/step 值
  - 控件类型（滑块、输入框等）
- Colors 部分（可选）：
  - 某些艺术作品需要颜色选择器
  - 某些艺术作品可能使用固定颜色
  - 某些艺术作品可能是单色的（不需要颜色控件）
  - 根据艺术作品的需要决定

**每件艺术作品都应具有独一无二的参数和算法！** 固定部分提供一致的 UX——其余所有内容表达独特愿景。

### 必需功能

**1. 参数控件**
- 用于数值参数（粒子数、噪声尺度、速度等）的滑块
- 用于调色板颜色的颜色选择器
- 参数更改时实时更新
- 用于恢复默认值的 Reset 按钮

**2. 种子导航**
- 显示当前种子编号
- 用于遍历种子的 Previous 和 Next 按钮
- 用于随机种子的 Random 按钮
- 用于跳转到特定种子的输入框
- 在请求时生成 100 个变体（种子 1–100）

**3. 单一制品结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js from CDN - always available -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* All styling inline - clean, minimal */
    /* Canvas on top, controls below */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- All parameter controls -->
  </div>
  <script>
    // ALL p5.js code inline here
    // Parameter objects, classes, functions
    // setup() and draw()
    // UI handlers
    // Everything self-contained
  </script>
</body>
</html>
```

**关键**：这是单一制品。没有外部文件，也不导入任何内容（p5.js CDN 除外）。所有内容均内联。

**4. 实现细节——构建侧边栏**

侧边栏结构：

**1. Seed（固定）**——始终完全按原样包含：
- 种子显示
- Prev/Next/Random/Jump 按钮

**2. Parameters（可变）**——为艺术作品创建控件：
```html
<div class="control-group">
    <label>Parameter Name</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
参数有多少个，就添加多少个 control-group div。

**3. Colors（可选/可变）**——如果艺术作品需要可调颜色，则加入此部分：
- 如果用户应能控制调色板，添加颜色选择器
- 如果艺术作品使用固定颜色，则跳过此部分
- 如果艺术作品为单色，则跳过

**4. Actions（固定）**——始终完全按原样包含：
- Regenerate 按钮
- Reset 按钮
- Download PNG 按钮

**要求**：
- 种子控件必须正常工作（prev/next/random/jump/display）
- 所有参数必须有 UI 控件
- Regenerate、Reset、Download 按钮必须正常工作
- 保留 Anthropic 品牌设计（UI 样式，而不是艺术作品的颜色）

2. **作为文件**：保存后在任何浏览器中打开——无需服务器
