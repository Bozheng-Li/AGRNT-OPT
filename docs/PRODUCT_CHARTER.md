# Product charter

## Mission

Build a continuously expanding, quality-first aggregation platform for Agent skills, plugins, MCP servers, official repositories, and other high-value open integrations. Growth has no artificial numeric ceiling, but every formally included project must be useful, traceable, legally compatible, and honestly represented.

## Formal inclusion contract

A project is formally integrated only when it has:

1. verified source identity and preserved provenance;
2. Chinese localization alongside original text;
3. capability, duplicate, license, security, and usefulness review;
4. a working runtime adapter or an explicitly evidenced remote integration;
5. a dedicated Web experience suited to the capability;
6. real verification whenever the required runtime conditions are available.

The system must never turn a marketing claim into a verified capability. Missing paid accounts, private credentials, special hardware, region access, or external availability must be represented as a concrete blocker.

## Lifecycle

`discovered → qualified → translated → adapted → web-ready → verified`

Side states are `blocked` and `deprecated`. A blocked entry retains its last proven lifecycle stage and a machine-readable reason. A deprecated entry remains traceable but is not publicly promoted.

## Web adaptation contract

Every formally integrated project gets its own route and task-oriented Web workflow. Integrations share the application shell, identity, conversations, permissions, logging, deployment, and base components, while each plugin defines its own inputs, settings, workflow, output renderer, examples, documentation, runtime state, and error messages. Complex capabilities require purpose-built interaction or visualization components.

## Completion model

This is a living platform, so success is not a one-time item count. The durable completion condition is a repeatable system that can discover, evaluate, translate, adapt, Web-enable, test, update, and retire projects without reducing quality or losing provenance.
